1. Use meaningful and descriptive variable/function/class names.
2. Follow PEP 8 style guide (indentation, line length, spacing).
3. Write modular code with functions and classes.
4. Add docstrings to modules, classes, and functions.
5. Use type hints for function signatures.
6. Handle exceptions properly.
7. Avoid global variables; use parameters and return values.
8. Write comments where the code is not self-explanatory.
9. Use list comprehensions and generator expressions where appropriate.
10. Keep code DRY (Don't Repeat Yourself) by reusing code.
